## 这是什么？

**Thinking Model Engine** 是一个 [OpenClaw](https://github.com/openclaw/openclaw) 技能（Skill），让 AI Agent 能像全科医生一样工作：

1. **接收你的问题** — "我该不该跳槽？""团队增长停滞了""总是拖延怎么办"
2. **诊断问题本质** — 从 10 类场景中匹配问题类型
3. **开具思维模型处方** — 从 117 个模型中精准选出 2-5 个最相关的
4. **输出可执行方案** — 不是列一堆理论，而是"明天你可以这样做"
5. **按需深挖** — 想要更深入？追加更多模型继续分析

### 不是什么？

- ❌ **不是思维模型库的电子版** — 那是 Obsidian 干的事
- ❌ **不是"帮我找个模型"的检索工具** — 那太浅了
- ✅ **是问题→分析→行动的完整引擎**

## 核心设计理念

### 处方 > 索引

传统做法：给你一堆模型让你自己选。
这个技能：**诊断 → 开处方 → 给药（行动方案）**。

### 按需加载，不爆 Context

117 个模型文件不会全部塞进 AI 的上下文窗口。每次只读取匹配到的 **2-5 个模型原文**，其余通过精简索引快速检索。

### 可信度分级

| 等级 | 含义 | 数量 |
|:----:|------|:----:|
| ⭐⭐⭐ | 经典理论，有坚实学术基础 | ~50 个 |
| ⭐⭐ | 业界广泛验证，实用框架 | ~45 个 |
| ⭐ | 启发性/自创工具，补充视角 | ~22 个 |

低可信度模型标注为"辅助视角"，不单独依赖。

### 深挖机制

每次分析的**最后一段必须是深挖邀请**：

> "如果你想深入分析 X 方面，我还可以调用 XX、XX、XX 模型继续展开。需要深挖吗？"

用户说"深挖"，Agent 追加 3-5 个模型在前次分析基础上深化。

## 快速开始

### 安装

```bash
# 方式一：Clone 到 OpenClaw skills 目录
git clone https://github.com/SakenW/thinking-model-engine.git \
  ~/.openclaw/workspace/skills/thinking-model-engine

# 方式二：通过 ClawHub 安装（如果已发布）
clawhub install thinking-model-engine
```

### 使用

安装后，直接对 Agent 说：

- "我在考虑要不要跳槽，帮我想想"
- "产品用户增长停滞了，分析一下"
- "我最近总是拖延，怎么办"
- "帮我用思维模型深入分析一下我的职业规划"

Agent 会自动触发本技能，完成全部分析。

## 输出格式示例

每次分析都遵循固定结构：

```
🔍 诊断     — 问题本质是什么
🧠 分析框架 — 用了哪些模型 & 为什么
📊 分析过程 — 用模型框架逐步展开
✅ 行动方案 — 具体可执行的步骤
⚠️ 陷阱提醒 — 常见认知偏误
🔗 深挖方向 — 还可以继续分析什么（必带）
```

## 技能架构

```
用户提问
   ↓
STEP 1: 判断是否适用（排除纯查询/执行/情绪宣泄/紧急救火）
   ↓
STEP 2: 加载 scenario-prescriptions.md → 匹配场景 P0-P9 → 选轻量/深度版
   ↓
STEP 3: 加载 model-quickref.md → 获取文件映射 → 只读 2-5 个模型 .md
   ↓
STEP 4: 组合分析 → 按 6 段式格式输出
   ↓
STEP 5: 用户说"深挖" → 加载 combo-patterns.md → 追加 3-5 个模型
   ↓
STEP 6: 可选对接 taskflow / GTD / Obsidian / Notion
```

## 模型来源

117 个思维模型来源于一个经过精心维护的 Obsidian 知识库，覆盖：

- **经典学术理论**：达克效应、前景理论、马斯洛需求层次、PDCA、Cynefin…
- **商业管理框架**：BCG 矩阵、波特五力、OKR、GTM、商业模式画布…
- **效率方法论**：GTD、番茄工作法、时间阻塞、艾森豪威尔矩阵…
- **心理学洞察**：卡尼曼双系统、心流、福格行为模型、峰终定律…
- **沟通表达**：金字塔原理、PREP、STAR 法则、SCQA…

每个模型文件包含：概述 → 核心内容 → 使用步骤 → 示例 → 注意事项 → 变体/相关模型 → 适用场景 → 模板。
