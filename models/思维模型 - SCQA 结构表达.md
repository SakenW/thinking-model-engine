# SCQA 结构表达 (SCQA Framework)

## 概述

芭芭拉·明托（《金字塔原理》）提出的一种叙事结构：**Situation（情景）→ Complication（冲突）→ Question（问题）→ Answer（回答）**。用"讲故事"的方式开场，快速建立共识并引出核心观点。

## 四个要素

| 要素 | 作用 | 写法 |
|------|------|------|
| **S - Situation 情景** | 建立共同认知背景 | 描述读者已知的、无争议的事实 |
| **C - Complication 冲突** | 打破平衡，制造张力 | 背景 + 什么出了问题/什么变了 |
| **Q - Question 问题** | 引导读者思考 | "那怎么办？""如何解决？" |
| **A - Answer 回答** | 给出核心观点/方案 | 你的结论或建议 |

## 公式

```
S：大家都认可的背景事实
C：但是，出现了问题/变化/挑战
Q：所以，我们该怎么办？
A：我认为应该 [核心观点]。
```

## 使用步骤

1. **想清楚你的 Answer** — 先有结论，再设计开头（SCQA 只是包装）
2. **写 S** — 读者已知的事实，2-3 句话
3. **写 C** — 打破 S 的平衡点，制造"不得不解决"的紧迫感
4. **写 Q** — 自然过渡，引导到你的 A
5. **展开 A** — 用金字塔原理展开论证

## 示例

> **汇报方案：建议升级客服系统**
>
> - **S**：目前我们每天处理约 500 个客户咨询，平均响应时间 2 小时
> - **C**：但上个月产品更新后咨询量暴增到 1200，客服团队加班也处理不完，客户满意度从 4.5 降到 3.2
> - **Q**：如何在不大量招人的情况下提升客服效率和客户满意度？
> - **A**：建议引入智能客服系统分流 60% 常见问题，预计可将响应时间降至 30 分钟，满意度回升到 4.0 以上

## 注意事项

- S 必须是读者**无争议**的事实——如果 S 就有分歧，C 就立不住
- C 是关键——**冲突越尖锐，读者越想看答案**
- Q 不一定要写出来——有时 C 本身就暗示了 Q
- 口头表达中 S 可以极短（一句话），书面可以稍详细

## 变体/相关模型

- **金字塔原理**：SCQA 所在的大框架
- **PREP 结构**：更简洁的表达框架（Point-Reason-Example-Point）
- **STAR 法则**：叙事框架（Situation-Task-Action-Result）
- **WSN 模型**：反思框架（What-So What-Now What）
- **故事弧线 (Story Arc)**：文学中的经典叙事结构

## 适用场景

- 汇报/PPT 开场
- 邮件/文档开头
- 演讲开场白
- 商业提案引言
- 面试自我介绍
- 说服性写作

## 模板

```markdown
## S - Situation（背景）
_

## C - Complication（冲突/变化）
_

## Q - Question（引出的问题）
_

## A - Answer（核心回答）
_

## 展开论证
- 论据 1：_
- 论据 2：_
- 论据 3：_
```
