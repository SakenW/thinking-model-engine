# 写在后面 (The Afterword Principle)

## 概述

**把最重要的信息放在结尾，让读者/听众带着你想要的核心信息离开。**

这不是一个传统意义上的"模型"，而是一个**表达与传播策略**——源于心理学中的**近因效应 (Recency Effect)**，即最后接收到的信息最容易影响判断和记忆。

## 核心原理

### 记忆曲线

```
注意力
  ↑
  │ ╲         ╱
  │   ╲      ╱
  │     ╲   ╱ ← 结尾（近因效应）
  │      ╲╱
  │
  └──────────────────→ 时间
  首因    中间    近因
  效应           效应
```

- **首因效应**：开头最容易被记住
- **近因效应**：结尾最容易影响决策
- **中间部分**：最容易被遗忘

## 应用场景

### 1. 写作
- 文章结尾点题升华，不要草草收尾
- 报告的"结论与建议"放在最后，且要最强有力
- 邮件最后一句话放行动号召（Call to Action）

### 2. 演讲/汇报
- 结尾的总结和呼吁是听众带走的东西
- Q&A 之前先做总结（避免 Q&A 冲淡核心信息）
- 最后一句要有力量

### 3. 谈判/沟通
- 把你的底线或最终提议放在最后说
- 结束语定调（"我们达成共识的核心是…"）

### 4. 产品设计
- 用户旅程的最后一步决定整体体验（结合峰终定律）
- 卸载/取消流程的体验影响品牌印象

## 实用技巧

| 技巧 | 做法 |
|------|------|
| **首尾呼应** | 开头提出的问题，结尾给出答案 |
| **行动号召** | 结尾明确告诉对方"下一步做什么" |
| **情绪收束** | 结尾调动目标情绪（希望/紧迫/信任）|
| **简洁有力** | 结尾越短越好，一句话最好 |

## 与其他模型的关系

- **近因效应**：心理学基础
- **峰终定律**：体验由峰值和结尾决定——"写在后面"就是控制"终"
- **PREP 结构**：最后的 P 就是"写在后面"
- **金字塔原理**：结论也可以放在最后（归纳式 vs 演绎式）
- **SCQA**：A（Answer）可以放在结尾

## 注意事项

- "写在后面"不等于"把重要的藏在最后"——开头也要有吸引力
- 长内容（报告/书）中，首因效应更重要
- 短交互（对话/邮件）中，近因效应更重要
- 如果中间有长时间间隔，近因效应会增强（因为工作记忆刷新了）
