# 峰终定律 (Peak-End Rule)

## 概述

由诺贝尔奖得主 **丹尼尔·卡尼曼** 提出。该定律指出：**人们对一段体验的记忆，主要由两个时刻决定：峰值（最强烈的瞬间，正向或负向）和结尾（结束时的感受）。** 过程的长度和其他时刻对记忆的影响微乎其微。

## 核心内容

### 核心公式

```
体验记忆 ≈ 峰值（最强瞬间）+ 结尾（最后感受）

⚠️ 注意：
- 过程总时长几乎不影响记忆
- 平均感受不重要
- 只有"最强烈的那个点"和"结束时怎么样"重要
```

### 经典实验：冷水实验

```
参与者将手放入冰水中（很痛苦）：

【A 组】冰水 60 秒
  痛苦曲线：████████████
              ↑60秒结束
  记忆：很痛苦

【B 组】冰水 60 秒 + 再加 15 秒温水（不那么痛）
  痛苦曲线：████████████░░░
                         ↑75秒结束
  记忆：比 A 组好！愿意再次参与

逻辑上 B 组更痛苦（多受了 15 秒），
但因为结尾更好，记忆反而更正面！
```

### 峰值与终值的类型

| 类型 | 定义 | 设计策略 |
|------|------|---------|
| **正峰值** | 体验中最好的时刻 | 刻意制造惊喜、超预期时刻 |
| **负峰值** | 体验中最差的时刻 | 消除或缓解痛点 |
| **正终值** | 结束时的好感受 | 在结尾留下好的最后印象 |
| **负终值** | 结束时的坏感受 | 避免在结尾出问题 |

## 使用步骤

### 设计体验（产品/服务/活动）

1. **识别关键触点** — 用户/参与者会经历哪些关键时刻？
2. **设计正峰值** — 在哪里制造"哇"的时刻？（惊喜、超越预期）
3. **消除负峰值** — 最痛的点在哪里？能不能减少或消除？
4. **设计正终值** — 结尾时怎么让对方带着好感觉离开？
5. **忽略过程时长** — 不要因为过程长就认为需要更多投入，关键是峰和终

### 回顾体验（自我评估）

1. **回忆峰值** — 这段经历中最强烈的时刻是什么？
2. **回忆结尾** — 结束时我的感受如何？
3. **意识到偏差** — 我的记忆是否被峰终绑架了？整体真的那么好/差吗？

## 示例

> **迪士尼乐园的峰终设计**
>
> - **正峰值**：坐过山车的最高点、烟花的绽放瞬间、和迪士尼角色合影
> - **负峰值**：排队 2 小时（但通过娱乐化排队缓解）
> - **正终值**：出口处的商店 + "下次见"的告别 → 带着快乐回忆离开
> - 排队再长，记忆中留下的还是过山车和烟花
>
> > **一次糟糕的离职面试**
> >
> > - 你在公司工作了 3 年，大部分时间还不错
> > - 但最后一个月的离职谈话中，HR 态度冷漠、流程繁琐
> > - **结果**：你对这家公司的最终记忆 = 糟糕的结尾 → 再也不推荐朋友来
> > - 3 年的好被最后 30 分钟毁掉了

## 注意事项

- 峰终定律描述的是**记忆**，不是**当下体验**——两者可以完全不同
- 负峰值比正峰值对记忆的影响更大（损失厌恶 again）
- **不要操纵峰终来掩盖实质性问题**——如果产品本身很差，再好的结尾也救不了长期口碑
- 在人际交往中，**最后一次互动的质量决定了关系的长期记忆**

## 变体/相关模型

- **近因效应 (Recency Effect)**：心理学基础之一，结尾信息权重更高
- **服务蓝图 (Service Blueprint)**：设计服务体验的工具，可结合峰终定律使用
- **客户旅程地图 (Customer Journey Map)**：识别峰终点的工具
- **Kano 模型**：区分基本需求、期望需求和兴奋需求（兴奋需求=制造正峰值）
- **NPS (Net Promoter Score)**：峰终定律直接影响推荐意愿

## 适用场景

- 产品/服务体验设计（用户旅程的关键时刻设计）
- 活动策划与会议设计
- 客户成功与留存策略
- 个人品牌管理（每次交互的结尾印象）
- 绩效反馈与面谈设计
- 旅游/餐饮/酒店的服务设计
