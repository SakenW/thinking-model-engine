# 系统思维 (Systems Thinking)

## 概述

一种**整体性、动态性、关系性**的思维方式：不把事物看成孤立的因果线，而是看作**互相影响的系统**，关注要素之间的关系、反馈回路和时间延迟。系统思维的经典："你无法通过把青蛙切成两半来研究它的活力。"

## 核心概念

| 概念 | 说明 | 示例 |
|------|------|------|
| **整体性** | 系统大于部分之和 | 团队不是个人能力的简单叠加 |
| **反馈回路** | 输出反过来影响输入 | 越卖越便宜 → 成本越低 → 更便宜（正反馈） |
| **时间延迟** | 行动和结果之间有时间差 | 今天招的人要 6 个月才能产出 |
| **存量与流量** | 存量是积累，流量是变化率 | 银行余额（存量）vs 收入支出（流量） |
| **非线性** | 输入输出不成正比 | 100% 努力可能只带来 20% 提升 |

## 两种反馈回路

```
正反馈（增强回路）          负反馈（调节回路）
     ↑                           ↑
   更多 → 更好 → 更多 → 更好    偏差 → 调整 → 回归
     │                           │
     └──────────→ ←─────────────┘
  （增长或崩溃）              （稳定或平衡）
```

## 使用步骤

1. **画出系统图** — 要素 + 关系箭头
2. **识别反馈回路** — 有哪些正/负反馈？
3. **找到杠杆点** — 在哪里小改动能产生大影响？
4. **考虑时间延迟** — 效果什么时候显现？
5. **避免线性思维** — 不要"头痛医头脚痛医脚"

## 示例

> **团队加班问题（系统视角）：**
>
> - 线性思维：大家加班 → 让他们别加班了
> - 系统思维：
>   - 需求太多 → 加班 → 疲劳 → 效率下降 → 需求积压更多 → 更多加班（**恶性正反馈**）
>   - 杠杆点不在"禁止加班"，而在**需求管理**或**招聘**
>
> **减肥的系统思维：**
> - 不是少吃多动（线性）
> - 而是：饮食 → 能量 → 运动 → 代谢 → 睡眠 → 饥饿感 → 饮食（**闭环系统**）

## 注意事项

- 系统思维容易变成"想太多"——**不要过度建模简单问题**
- 杠杆点往往不直观——**直觉上最明显的地方往往不是真正的杠杆点**
- 画图比空想有效——用纸笔画出关系
- 系统边界的选择本身就是一种判断

## 变体/相关模型

- **冰山模型 (Iceberg Model)**：系统思维的层次框架（事件→模式→结构→心智模式）
- **因果环路图 (Causal Loop Diagram)**：系统思维的可视化工具
- **二阶思维**：考虑时间延迟的简化版系统思维
- **飞轮效应**：正反馈回路的商业应用
- **瓶颈理论 (TOC)**：寻找系统约束的方法

## 适用场景

- 复杂问题分析（组织、生态、社会问题）
- 产品设计中的系统性 Bug 排查
- 战略规划（考虑连锁反应）
- 个人成长系统设计
- 根因分析（超越 5 Whys 的表面原因）
