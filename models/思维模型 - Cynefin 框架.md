# Cynefin 框架 (Cynefin Framework)

## 概述

**判断你所面对的问题属于哪种类型，然后用匹配的方法去解决。**

由 Dave Snowden 提出，是处理复杂性和不确定性的**决策导航工具**。"Cynefin"（肯尼芬）是威尔士语，意为"栖息地"或"多重归属之地"。

## 五个领域

```
         混沌 (Chaotic)
       ┌─────────────┐
       │  快速行动    │
       │  稳定→转复杂  │
       ├─────────────┤
复杂    │             │  繁杂
(Complex)│   无序      │ (Complicated)
       │  分解→归入四域 │
       ├─────────────┤
       │  最佳实践    │
       │  感知-分类-响应│
       └─────────────┘
           简单 (Clear/Simple)
```

### 1. 简单/清晰 (Clear)
**特征：** 因果关系明确，最佳实践已知。
- **方法：** 感知 → 分类 → 响应（按已有流程做）
- **危险：** 过度自信，认为所有问题都简单
- **例子：** 走流程报销、标准客服问答
- **⚠️ 坠落风险：** 环境变化后，简单域可能变成混乱域

### 2. 繁杂/复合 (Complicated)
**特征：** 因果关系存在但不易发现，需要专家分析。
- **方法：** 感知 → 分析 → 响应（找专家、做分析）
- **危险：** 专家可能意见不同，分析可能过度
- **例子：** 技术架构选型、复杂业务流程优化
- **工具：** 专家判断、系统分析、决策矩阵

### 3. 复杂 (Complex)
**特征：** 因果关系只能**事后回溯**发现，无法提前预测。涌现行为。
- **方法：** 探测 → 感知 → 响应（试错、实验、安全失败）
- **关键：** 不要试图控制，要创造条件让答案涌现
- **例子：** 新产品市场反应、团队文化、社交网络传播
- **工具：** MVP、A/B 测试、假设驱动

### 4. 混沌 (Chaotic)
**特征：** 没有可辨识的因果关系，一切混乱。
- **方法：** 行动 → 感知 → 响应（先稳住局面再说）
- **关键：** 快速行动比分析重要
- **例子：** 突发危机、系统崩溃、公关灾难
- **工具：** 指挥控制、应急响应

### 5. 无序 (Disorder)
**特征：** 不知道自己处于哪个域。
- **方法：** 先分解问题，判断属于上面四个域中的哪一个
- **危险：** 用错误域的方法解决问题（如用"简单"的方法处理"复杂"问题）

## 核心决策逻辑

| 你面对的情况 | 应该怎么做 |
|-------------|----------|
| 答案已知 | 直接执行最佳实践（简单域）|
| 需要专家 | 分析后执行（繁杂域）|
| 答案未知但可以探测 | 小实验、快速试错（复杂域）|
| 完全失控 | 先稳定，再转移（混沌域）|
| 不确定属于哪种 | 先归类，再选方法（无序→四域）|

## 最常见的错误

| 错误 | 表现 |
|------|------|
| **把复杂当繁杂** | 请专家分析后给方案（但复杂问题没有"正确"答案）|
| **把简单当复杂** | 过度设计、过度分析（本来按流程做就行）|
| **把混沌当复杂** | 慢慢做实验（但混沌需要立刻行动）|
| **永远留在混沌域** | 每次都靠"英雄主义"救火 |

## 适用场景

- 判断项目/问题该用什么方法
- 组织变革的策略选择
- 产品决策（需求是简单还是复杂？）
- 危机管理分级
- 个人面对困难时的策略选择

## 与其他模型的关系

- **系统思维**：复杂域是系统思维的主要应用场景
- **OODA 循环**：混沌域需要快速 OODA
- **PDCA**：繁杂域和简单域适合 PDCA
- **精益创业**：复杂域的最佳实践
- **第一性原理**：繁杂域的分析工具

## 📌 2024 年更新（Dave Snowden 原作者）

> Snowden 在 2024 年的年度框架回顾中承认，他在 Cynefin 中使用"混沌 (Chaos)"一词可能**过于自信地假设了其含义的清晰度**。他探讨了该词的多种不同但连贯的含义：
>
> - **数学混沌**：确定性非线性系统中的敏感依赖初始条件（与 Cynefin 的"无因果关系"含义不同）
> - **日常语义的混乱**：完全失控、无序的状态（更接近 Cynefin 本意）
> - **哲学/本体论层面的混沌**：前结构化的原始状态
>
> **对我们的启示：** 使用 Cynefin 时，"混沌域"应理解为"没有可辨识的因果关系、需要先建立秩序"的状态，而非数学意义上的混沌理论。在实际应用中，这个区分更多是学术精确性的问题——操作层面"快速行动稳定局面"的建议仍然有效。
