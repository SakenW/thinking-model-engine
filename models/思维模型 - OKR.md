# OKR (Objectives and Key Results)

## 概述

**目标与关键结果** — Intel 创始人 Andy Grove 发明，Google 推广的目标管理框架。

- **O (Objective)**：方向性的、鼓舞人心的**目标**
- **KR (Key Results)**：可衡量的**关键结果**，用于判断 O 是否达成

## 结构

```
Objective (O): 用一句激动人心的话描述我们要去哪里
├── KR 1: 可衡量的结果（数字！）
├── KR 2: 可衡量的结果（数字！）
└── KR 3: 可衡量的结果（数字！）
```

## 好的 O 和 KR 的标准

### 好的 Objective
- 定性的、方向性的
- 简短、有感染力
- 时间范围明确（通常季度）
- **不是** KPI 的汇总

> ✅ "成为开发者最喜爱的 AI 平台"
> ❌ "提高各项指标 10%"

### 好的 Key Result
- **必须是量化的**（数字或明确的二元状态）
- 有挑战性（达成概率 60-70%）
- 结果导向，不是待办事项清单
- 通常 2-5 个 per O

> ✅ "日活用户从 5 万提升到 15 万"
> ❌ "发布新版本" （这是任务，不是 KR）

## 打分与复盘

| 分数 | 含义 | 处理 |
|------|------|------|
| 1.0 (100%) | 超额完成 | O 设得太低了 |
| 0.7 (70%) | 达成 — 理想状态 | 难度适中，继续推进 |
| 0.4 (40%) | 有进展但没达成 | 分析原因，调整策略 |
| 0.0 (0%) | 没有进展 | 方向可能错了 |

## 与其他框架的关系

| | OKR | KPI | SMART | MBO |
|---|------|-----|-------|-----|
| 目的 | 对齐 + 聚焦 | 绩效考核 | 个人目标设定 | 管理工具 |
| 频率 | 季度 | 年/月 | 不定 | 年度 |
| 公开 | 全员公开 | 可能保密 | 个人 | 上下级 |
| 与薪酬挂钩 | ❌ 不挂钩 | ✅ 挂钩 | 视情况 | ✅ 挂钩 |

## 实施建议

1. **从少开始**：每人/每团队最多 3-5 个 O
2. **对齐先于数量**：确保上下级 O 对齐
3. **双周 Check-in**：不要等到季末才发现跑偏了
4. **区分承诺型 OKR 和挑战型 OKR**

## 常见误区

| 误区 | 真相 |
|------|------|
| "OKR 就是 KPI 的另一种叫法" | OKR 对齐方向，KPI 考核绩效，目的不同 |
| "KR 没达成就是失败" | 0.6-0.7 是理想状态，说明目标有挑战性 |
| "OKR 要和薪酬挂钩" | 挂钩后大家会设保守目标，失去 OKR 的意义 |
| "每个 O 都必须有 5 个 KR" | 2-4 个 KR 就够了，质量 > 数量 |
| "OKR 只适合互联网公司" | 任何需要目标对齐的组织都适用 |

## 适用场景

- 公司/团队季度目标对齐
- 个人目标管理与聚焦
- 创业公司里程碑规划
- 项目优先级排序
- 跨部门协作对齐

## 变体/相关模型

- **KPI**：绩效考核工具，与 OKR 互补而非替代
- **SMART 原则**：KR 设定时的检验框架
- **BSC 平衡计分卡**：战略层面的目标管理
- **MBO 目标管理**：Peter Drucker 的经典管理框架
- **OGSM**：战略-目标-策略-度量四层分解
