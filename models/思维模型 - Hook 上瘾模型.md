# Hook 上瘾模型 (Hook Model)

## 概述

Nir Eyal 在《上瘾》中提出的用户行为设计框架：通过 **Trigger（触发）→ Action（行动）→ Variable Reward（多变酬赏）→ Investment（投入）** 四步循环，让用户形成习惯性使用产品。

## Hook 循环

```
  ┌──────────────────────────────┐
  │                              │
  ▼                              │
┌─────────┐    ┌──────────┐    ┌───────────┐    ┌──────────┐
│ Trigger │ → │  Action   │ → │   Reward   │ → │Investment│
│  触发   │    │   行动   │    │  多变酬赏  │    │   投入   │
└─────────┘    └──────────┘    └───────────┘    └────┬─────┘
     ▲                                              │
     └──────────────────────────────────────────────┘
                    下一次触发的触发器
```

## 四个阶段详解

| 阶段 | 说明 | 关键问题 | 示例 |
|------|------|---------|------|
| **Trigger 触发** | 促使用户行动的刺激 | 外部触发？内部触发？ | 微信通知（外部）/ 无聊时想刷（内部） |
| **Action 行动** | 用户为获得酬赏做的最简行为 | 行动是否足够简单？ | 下拉刷新、点击"喜欢" |
| **Reward 酬赏** | 用户获得的反馈，**关键：多变** | 猎人的搜寻感？部落的社会感？自我的掌控感？ | 刷到不同内容（抖音）、点赞数变化 |
| **Investment 投入** | 用户在产品中的累积投入 | 数据/内容/关注/声誉/技能 | 发了朋友圈、关注了人、训练了算法推荐 |

## 使用步骤

1. **定义内部触发** — 用户在什么情绪/场景下想到你的产品？
2. **设计外部触发** — 用什么把用户拉回来？（推送、邮件、社交邀请…）
3. **简化行动** — 减少摩擦到极致（3 秒法则）
4. **设计多变酬赏** — 不要每次都一样，制造"期待感"
5. **设计投入点** — 让用户的下一次使用更有价值

## 示例

> **抖音的 Hook 设计：**
>
> - **Trigger**：无聊（内部）+ 推送通知（外部）
> - **Action**：打开 App → 自动播放视频（零操作）
> - **Reward**：每个视频都不同——有的好笑、有的惊艳、有的无趣（**多变**是关键）
> - **Investment**：点赞→算法更准；关注→更多内容；自己拍→社交资本
>
> → 投入越多，下次触发越强 → 循环加深

## 注意事项

- Hook 模型的伦理边界：**用于帮助用户养成好习惯 vs 利用成瘾机制**
- "多变酬赏"是最核心的设计要素——固定奖励很快会厌倦
- 投入不是让用户"花钱"，而是**让产品因用户的使用而变得更好用**
- 不是所有产品都需要 Hook——工具型产品可能只需要效率

## 变体/相关模型

- **福格行为模型 (Fogg Behavior Model)**：B=MAP，Hook 的理论基础之一
- **双钻模型 (Double Diamond)**：产品设计流程，Hook 是其中行为设计部分
- **游戏化 (Gamification)**：酬赏设计的具体方法论
- **心流理论**：Reward 阶段的理想状态

## 适用场景

- 产品用户留存设计
- 习惯养成类 App 开发
- 内容平台运营策略
- 社交产品功能设计
- 个人习惯培养（正向使用）

## 📌 学术与行业争议（2026 更新）

> **Hook 模型是科学还是营销话术？**
>
> 2026 年 The Behavioral Scientist 发表深度批评文章，指出：
> - Hook 模型的四步循环**缺乏严格的实证研究支持**——Eyal 的书主要基于案例观察和商业经验，而非对照实验
> - "内部触发"等核心概念**难以操作化测量**——无法可靠地识别和验证
> - 许多"成功案例"存在**幸存者偏差**——用 Hook 分析成功产品容易，但用 Hook 设计新产品未必能复制成功
> - **伦理问题**：将行为设计技术系统化地用于增加用户成瘾性，引发广泛担忧
>
> **平衡视角：** Hook 模型作为**产品分析的启发式框架**仍有价值（帮助理解为什么某些产品让人上瘾），但不应被视为经过严格验证的科学理论。使用时建议：
> 1. 作为**事后分析工具**而非**设计公式**
> 2. 结合其他行为科学理论（如福格行为模型 B=MAP）交叉验证
> 3. 始终进行**伦理自检**（模板中已包含）

## 模板

```markdown
## 产品/习惯：_

### Trigger（触发）
- 外部触发：_
- 内部触发（核心）：_

### Action（行动）
- 核心行动：_
- 摩擦点 & 简化方案：_

### Reward（酬赏）
- 酬赏类型：□猎物 □部落 □自我
- 多变性设计：_

### Investment（投入）
- 用户投入什么：_
- 投入如何提升下次体验：_

### 伦理自检
- 这个 Hook 对用户有益吗？✅/⚠️
```
