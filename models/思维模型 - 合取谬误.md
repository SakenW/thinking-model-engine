# 合取谬误 (Conjunction Fallacy)

## 概述

一种常见的认知偏差：人们倾向于认为**更具体、细节更多的描述**比更宽泛的描述**更有可能为真**，即使从逻辑上讲，具体事件（A∩B）的概率不可能高于其组成部分（A 或 B）。

## 经典实验：Linda 问题

> Linda 31 岁，单身，直言不讳，主修哲学。学生时代关注歧视和社会正义问题，参加过反核游行。
>
> **请按可能性排序：**
>
> A. Linda 是银行出纳员
> B. Linda 是银行出纳员 **且** 积极参与女权运动
>
> **结果：约 85% 的人认为 B 更可能** ❌
>
> **逻辑分析：**
> - P(银行出纳员且女权主义者) ≤ P(银行出纳员)
> - 因为"银行出纳员且女权主义者"是"银行出纳员"的**子集**
> - 但细节让故事更"合理"，所以直觉上觉得更可能

## 为什么会犯错？

| 原因 | 说明 |
|------|------|
| **代表性启发 (Representativeness)** | 细节越多越像我们脑海中的"典型"，感觉越真实 |
| **叙事偏好** | 大脑喜欢好故事胜过抽象概率 |
| **缺乏概率思维训练** | 教育系统很少教集合论和概率逻辑 |

## 使用步骤

1. **识别判断场景** — 你在评估某个事件/人的可能性吗？
2. **检查是否有"额外细节"** — 这些细节是否缩小了范围？
3. **用逻辑检验** — "A 且 B"是否比"A 或 B"或单独的"A"更具体？
4. **去掉细节重新评估** — 剥离细节后概率是变高还是变低？
5. **用数字辅助** — 如果能估算概率，写下来比较

## 示例

> **投资决策中的合取谬误：**
>
> - 方案 A：这家公司明年营收增长 15%
> - 方案 B：这家公司明年营收增长 15% **且** 成功进入东南亚市场**且** 推出的新产品获得好评
>
> → 很多人选 B，因为听起来更完整。但 B 的概率一定低于 A。
>
> **招聘中的合取谬误：**
>
> - "我们要招一个 Python 熟练、有 AI 经验、做过 ToB 产品、沟通能力强、年龄 30 左右、接受降薪的人"
>
> → 每多一个条件，候选人池就缩小一圈。5 个条件可能把概率降到接近零。

## 注意事项

- 合取谬误在**专业领域**中反而更容易出现——专家更容易被"合理的故事"误导
- 在风险评估、投资决策、战略规划中危害最大——**这些场景容不得概率错误**
- 对抗方法不是消除直觉，而是**建立"概率检查"的习惯**

## 变体/相关模型

- **基础率忽视 (Base Rate Neglect)**：相关偏差——忽略先验概率
- **框架效应 (Framing Effect)**：同一信息不同表述导致不同判断
- **卡尼曼双系统 (System 1/System 2)**：合取谬误是 System 1 快思维的产物
- **贝叶斯思维 (Bayesian Thinking)**：正确的概率更新方式
- **奥卡姆剃刀**：与合取谬误互补——简单解释优先

## 适用场景

- 投资与金融决策
- 风险评估
- 招聘与人选评估
- 战略规划（避免过度具体的假设）
- 法律判决（证据评估）
- 新闻与信息辨别
- 日常概率判断
