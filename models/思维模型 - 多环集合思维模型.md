# 多环集合思维模型 (Multi-Ring Set Thinking)

## 概述

一种用**多个相交的圆环（韦恩图的扩展）**来分析复杂关系的思维工具。不同于二环（A∩B），多环模型可以同时展示 3 个及以上要素之间的**交集、并集、独立区域和空缺**，适合分析"多方博弈""多维平衡"类问题。

## 基本结构

```
三环模型：          四环模型：
   A                 A
  ╱ ╲              ╱ ╲
 /     \    B      /───\
│   ABC  │        │ ABD │
 \  BC  /         │  CD  │
  ╲ ╱            \───/
   C                C
                  D

关键区域含义：
- 只在 A 中：A 独有特征
- A∩B：A 和 B 的交集（共同点）
- A∩B∩C：三者核心交集（最稀缺/最有价值）
- 外部：都不包含的区域（盲区/机会）
```

## 使用步骤

1. **确定参与要素** — 需要分析几个维度/方？（2-5 个为宜）
2. **画圆环** — 每个圆代表一个要素，大小可以不同
3. **标注各区域** — 每个交集/独立区意味着什么？
4. **找到核心洞察** — 最有价值的区域是哪个？最危险的空白在哪里？
5. **导出行动** — 如何向核心交集移动？如何填补空白？

## 示例

> **理想工作的三环模型：**
>
> - A = 我擅长的
> - B = 我喜欢的
> - C = 市场愿意付费的
>
> | 区域 | 含义 | 行动 |
> |------|------|------|
> | A∩B∩C | **理想工作** → 全力追求 | 目标 |
> | A∩B 但不在 C | "自嗨型爱好" → 不要当主业 | 业余做 |
> | A∩C 但不在 B | "能赚钱但痛苦的工作" → 当跳板 | 过渡 |
> | B∩C 但不在 A | "想做但能力不够" → 重点学习方向 | 投资 |
> | 只有 A | "无用技能" → 寻找应用场景 | 探索 |
>
> **产品成功的三环模型（Ikigai 变体）：**
>
> - 用户需求 × 技术可行 × 商业可行 = 产品市场契合 (PMF)

## 注意事项

- 圆的数量不要超过 5 个——**太多环看不懂**
- 圆的大小可以有差异——**不代表某些要素不重要**
- 核心价值往往在**多环交集处**——但那里也最难到达
- 有时最大的洞察来自**外部空白区**——"三个圆都没覆盖到的地方"

## 变体/相关模型

- **韦恩图 (Venn Diagram)**：数学基础工具
- **Ikigai（生之意义）**：日本四环模型（喜欢、擅长、世界需要、能赚钱）
- **产品市场契合 (PMF)**：三环模型的商业版本
- **SWOT 分析**：四象限版的多维分析
- **3C 战略三角**：三环的商业战略应用

## 适用场景

- 职业规划（兴趣×能力×市场）
- 产品定位（用户×技术×商业）
- 团队协作分析（目标×能力×资源）
- 关系处理（自己需求×对方需求×现实约束）
- 决策权衡（收益×成本×风险）
- 创意寻找（可行性×原创性×价值）
