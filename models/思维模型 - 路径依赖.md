# 路径依赖 (Path Dependence)

## 概述

历史经济学核心概念：**当下的选择受过去选择的约束**，即使存在更优的替代方案，由于转换成本、习惯、既得利益等因素，系统会"锁定"在次优路径上。QWERTY 键盘就是经典例子——并非最优布局，但全世界都在用。

## 三种类型

| 类型 | 说明 | 例子 |
|------|------|------|
| **弱路径依赖** | 早期选择影响后期，但可逆转 | 编程语言选型，重构成本可控 |
| **强路径依赖** | 锁定效应明显，逆转代价极高 | 操作系统生态（Windows/macOS） |
| **锁定 (Lock-in)** | 几乎不可逆，只能等待颠覆性技术 | 基础设施标准（铁轨宽度） |

## 核心机制

```
早期随机/偶然选择
       ↓
  正反馈强化（用的人越多→越完善→更多人用）
       ↓
  转换成本递增（学习成本、兼容性、沉没投资）
       ↓
  锁定在次优路径上
```

## 使用步骤

1. **识别当前路径** — 我们现在走的路是怎么来的？
2. **评估锁定程度** — 还能掉头吗？代价多大？
3. **寻找解锁机会** — 技术变革？范式转移？新市场？
4. **如果无法解锁** — 如何在现有路径上做到最优？
5. **对新决策保持警觉** — 这个选择会不会制造新的路径依赖？

## 示例

> **公司技术栈评估：**
>
> - 当前：基于 Java 的单体架构（10 年积累）
> - 路径依赖来源：团队技能、已有代码库、运维工具链
> - 锁定程度：高——迁移到 Go/微服务需要 2 年+
> - 策略：不完全重写，但新服务采用新技术，渐进式替换

## 注意事项

- 路径依赖不等于错误——**有些路径依赖是合理的效率优化**
- 不要把所有"坚持"都当成路径依赖陷阱
- 打破路径依赖通常需要**外部冲击**或**颠覆性创新**
- 个人层面：职业路径、技术栈、思维模式都存在路径依赖

## 变体/相关模型

- **沉没成本谬误**：路径依赖的心理驱动因素之一
- **QWERTY 效应**：路径依赖的经典案例名称
- **技术锁定的制度分析**：Paul David 的学术框架
- **第一性原理**：打破路径依赖的思维工具——回到基本事实重新思考

## 适用场景

- 技术选型与架构决策评估
- 企业战略转型分析
- 行业标准竞争研究
- 个人职业规划（是否该转行）
- 产品设计（用户习惯迁移成本）
