# T型能力模型 (T-Shaped Skills)

## 概述

**既有广泛的知识面（横向），又在某一领域有深度（纵向）。**

字母" T "形象地描述了这种能力结构：横杠代表广度，竖杠代表深度。

## 能力结构

```
        深度（专长）
         │
    ┌────┤
    │    │
    │    │
────┴────┴────
  广度（通用能力）
```

| 维度 | 含义 | 内容 |
|------|------|------|
| **横杠（广度）** | 跨领域的通用知识和技能 | 沟通、项目管理、基础技术、商业意识、用户思维 |
| **竖杠（深度）** | 某一个领域的专精 | 前端开发、数据分析、产品设计、内容创作… |

## 为什么 T 型比 I 型好

| 类型 | 形状 | 特征 | 局限 |
|------|------|------|------|
| **I 型** | │ | 只有一项深度技能 | 沟通/协作困难，视野窄 |
| **T 型** | ├─ | 深度 + 广度 | 最理想的能力结构 |
| **一字型** | ─ | 什么都知道一点 | 没有核心竞争力 |
| **π 型** | ├┤ | 两个深度领域 | 更强但更难达到 |

## 实际应用

### 个人发展
1. 先建立**一项深度技能**（竖杠）— 这是你的立身之本
2. 逐步拓展**跨领域知识**（横杠）— 提升协作和视野
3. 深度决定你的**价值**，广度决定你的**适应性**

### 团队组建
- 理想团队 = 一群 T 型人的组合
- 每个人有专精，但能理解其他角色
- **避免**：全员 I 型（协作成本高）或全员一字型（没有专家）

### 招聘评估
- 不只看候选人"会不会做这个岗位的事"
- 还要评估"能不能理解其他岗位的需求"
- T 型人更容易融入跨职能团队

## 深度和广度的平衡

```
职业阶段          建议
─────────────────────────────
初级 (0-3年)  →  先 I 后 T（先深后广）
中级 (3-7年)  →  T 型（一深多广）
高级 (7年+)   →  π 型或梳型（多深多广）
```

## 注意事项

- **先有深度，再拓广度**——不要一开始就想什么都学
- 广度不是"什么都懂一点皮毛"，而是"能和其他领域的专家有效沟通"
- T 型是**动态的**——行业变化时，竖杠可能需要换方向
- 和达克效应配合理解：知道自己"广度"的边界很重要

## 与其他模型的关系

- **能力圈**：竖杠就是你的能力圈
- **达克效应**：T 型人更清楚自己不知道什么
- **学习型组织**：组织需要 T 型人来连接不同领域
- **第一性原理**：深度领域的底层理解方式
