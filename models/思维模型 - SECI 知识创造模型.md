# SECI 知识创造模型 (SECI Model)

## 概述

野中郁次郎和竹内弘高于 1995 年提出：知识创造通过**隐性知识 (Tacit)** 和**显性知识 (Explicit)** 之间的四种转换模式循环进行。**SECI = Socialization（社会化）→ Externalization（外化）→ Combination（组合）→ Internalization（内化）**。

## 两种知识

| 类型 | 特征 | 例子 |
|------|------|------|
| **隐性知识** | 难以言传、基于经验的、个人化的 | "怎么做美味红烧肉的火候感" |
| **显性知识** | 可以编码、存储、传递的 | "红烧肉菜谱步骤 1-10" |

## SECI 四种转换

| 模式 | 转换方向 | 机制 | 例子 |
|------|---------|------|------|
| **S - Socialization** 社会化 | 隐性 → 隐性 | 观察、模仿、师徒制 | 新人跟着老员工学"怎么和客户聊天" |
| **E - Externalization** 外化 | 隐性 → 显性 | 隐喻、类比、写作、复盘文档 | 老销售把客户沟通技巧写成 SOP |
| **C - Combination** 组合 | 显性 → 显性 | 整合、分类、重构现有知识 | 把多个团队的 SOP 整合成统一手册 |
| **I - Internalization** 内化 | 显性 → 隐性 | 实践、练习、做中学 | 新人读了 SOP 后在实际对话中变成自己的能力 |

## 循环图

```
     隐性知识          显性知识
       ↑                  │
       │    Socialization  │  Externalization
       │    (师徒制/观察)   │  (写文档/表达)
       │                  │
  ─────┤                  ├─────
       │                  │
       │  Internalization │  Combination
       │    (实践/练习)     │  (整合/重组)
       │                  ↓
     隐性知识          显性知识
```

## 使用步骤

1. **Socialization** — 创造交流机会（师徒制、社区、分享会）
2. **Externalization** — 鼓励知识输出（写文档、做分享、录视频）
3. **Combination** — 建立知识管理系统（Wiki、知识库、标签体系）
4. **Internalization** — 设计实践场景（练习、项目实战、复盘）

## 示例

> **团队知识管理实践：**
>
> - **S**：每月一次"开放日"，不同岗位互相观察学习
> - **E**：鼓励每个项目结束后写复盘文档，沉淀经验
> - **C**：建立内部 Wiki，把分散的文档整合分类
> - **I**：新人入职时先读 Wiki，再配导师带 2 周，在实际项目中内化

## 注意事项

- **隐性知识是组织最有价值的资产**——但它最容易被忽视
- Externalization 是最难的环节——"我知道但说不出来"是常态
- 知识管理工具解决的是 C（组合），但 S 和 E 需要文化支撑
- 不要只做 C（建知识库）而忽略 S 和 I（知识流动和实践）

## 变体/相关模型

- **费曼学习法**：E（外化）的具体操作方法
- **实践社群 (CoP)**：S（社会化）的组织形式
- **知识图谱**：C（组合）的可视化工具
- **卡片盒笔记法**：个人版的 SECI 系统

## 适用场景

- 组织知识管理建设
- 团队经验传承
- 新人培养体系设计
- 企业内部知识库运营
- 个人知识体系搭建
- 创新（新知识 = SECI 循环的产物）
