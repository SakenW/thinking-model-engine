# 六顶思考帽 (Six Thinking Hats)

## 概述

爱德华·德·波诺 (Edward de Bono) 提出的**平行思维工具**，用六种颜色的"帽子"代表六种思维模式，让团队讨论从混乱变得结构化。

## 六顶帽子

| 帽子 | 颜色 | 思维模式 | 关键词 | 规则 |
|------|------|----------|--------|------|
| ⚪ **事实帽** | 白色 | 客观事实和数据 | 数据、信息、已知、未知 | 只说事实，不发表观点 |
| 🔴 **情感帽** | 红色 | 直觉和感受 | 感觉、情绪、直觉 | 不需要理由，直觉就是答案 |
| ⚫ **风险帽** | 黑色 | 谨慎和批判 | 风险、问题、为什么不行 | 逻辑批判，找漏洞 |
| 🟡 **乐观帽** | 黄色 | 积极和价值 | 好处、优势、可行性 | 积极思考，找价值 |
| 🟢 **创意帽** | 绿色 | 创新和替代 | 新想法、替代方案、变化 | 打破常规，发散思维 |
| 🔵 **管理帽** | 蓝色 | 过程控制 | 总结、决策、下一步 | 控制流程，不参与内容 |

## 使用方法

### 方法 1: 序列使用（推荐）
按特定顺序依次戴帽，确保每个角度都覆盖到：

**场景：评估一个新想法**
```
🔵 蓝帽 → 定义讨论目标
⚪ 白帽 → 摆出所有相关事实和数据
🟡 黄帽 → 找好处和机会
⚫ 黑帽 → 找风险和问题
🟢 绿帽 → 提出替代方案和创新思路
🔴 红帽 → 表达直觉感受
🔵 蓝帽 → 总结和决策
```

### 方法 2: 单独使用
针对某个具体目的只戴一顶帽子：
- 需要快速反馈？→ 🔴 红帽
- 需要风险评估？→ ⚫ 黑帽
- 需要头脑风暴？→ 🟢 绿帽

## 关键规则

1. **同一时间只能戴一顶帽子** — 这是核心
2. **必须按帽子角色发言** — 戴绿帽时不能批评
3. **蓝帽主持人负责切换** — 不然容易乱
4. **每顶帽子的时间要控制** — 建议 2-5 分钟/帽

## 适用场景

- 会议讨论（避免各说各话）
- 决策评估
- 产品评审
- 团队冲突调解
- 个人思考（自己轮流戴）

## 注意事项

- 不要跳过黑帽 — 批判性思维是必要的
- 红帽不要过度使用 — 直觉不能代替分析
- 绿帽之后通常接黄帽或黑帽（先发散再收敛）
