# 马斯洛需求层次 (Maslow's Hierarchy of Needs)

## 概述

马斯洛于 1943 年提出的需求理论：人类需求像阶梯一样从低到高排列，**低层需求满足后才会追求高层需求**。虽然学术上有争议，但作为理解和分析人类动机的模型仍然非常实用。

## 五层需求

```
        ┌─────────────────┐
        │   ⑤ 自我实现     │  发挥潜能、追求意义
        │   Self-         │
        │   actualization  │
        ├─────────────────┤
        │   ④ 尊重需求     │  地位、认可、成就感
        │   Esteem         │
        ├─────────────────┤
        │   ③ 社交需求     │  归属感、爱、友谊
        │   Love/Belonging │
        ├─────────────────┤
        │   ② 安全需求     │  安全、稳定、保障
        │   Safety         │
        ├─────────────────┤
        │   ① 生理需求     │  食物、水、睡眠、住所
        │   Physiological  │
        └─────────────────┘
```

## 各层的关键洞察

| 层次 | 核心驱动力 | 满足时的状态 | 缺失时的状态 |
|------|-----------|------------|------------|
| **生理** | 生存本能 | 安心 | 恐慌、绝望 |
| **安全** | 稳定与可预期 | 放松 | 焦虑、警惕 |
| **社交** | 归属与连接 | 温暖、信任 | 孤独、疏离 |
| **尊重** | 能力与认可 | 自信 | 自卑、无力 |
| **自我实现** | 成长与意义 | 充实、愉悦 | 空虚、迷茫 |

## 使用步骤

1. **分析对象当前在哪个层次** — 个人/用户/员工
2. **判断需求缺口** — 哪一层没有被满足？
3. **针对性地满足** — 不要跳层（安全没满足时谈自我实现是无效的）
4. **动态观察** — 需求层次会随时间变化

## 示例

> **用户产品设计：**
>
> - 基础 App（生理/安全层）：稳定不崩溃、数据不丢失、隐私保护
> - 社交功能（社交层）：社区、互动、好友
> - 成就系统（尊重层）：等级、徽章、排行榜
> - 创作平台（自我实现层）：让用户创作和分享自己的内容

> **员工激励：**
>
> - 薪资够生活（①②）
> - 团队氛围好（③）
> - 被认可和尊重（④）
> - 有成长空间和意义感（⑤）

## 注意事项

- 不是严格的"一层满足完再上一层"——**可以同时追求多层**
- 不同人的层次跳跃不同——有人安全层没完全满足就追求自我实现
- 文化差异——**集体主义文化中社交层可能比尊重层更优先**
- 现代修订版增加了"认知需求"和"审美需求"

## 变体/相关模型

- **ERG 理论**：Alderfer 的简化版（生存-关系-成长）
- **双因素理论 (Herzberg)**：保健因素 vs 激励因素
- **自我决定理论 (SDT)**：自主、胜任、归属三大内在需求
- **Ikigai**：日本四维人生意义模型
- **需求月牙锯**：产品机会分析中的需求层次应用

## 适用场景

- 用户需求分析
- 员工激励设计
- 产品功能规划
- 营销策略制定
- 个人成长规划
- 社会问题分析
