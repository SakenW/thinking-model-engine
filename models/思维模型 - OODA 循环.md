# OODA 循环 (OODA Loop)

## 概述

**观察→判断→决策→行动的快速决策循环。** 谁的循环更快，谁就赢。

由美国空军上校约翰·博伊德（John Boyd）提出，最初用于空战战术，后被广泛用于商业战略、军事决策和应急响应。

## 四个阶段

```
  Observe（观察）
      │
      ▼
  Orient（判断）← 核心阶段
      │
      ▼
  Decide（决策）
      │
      ▼
   Act（行动）
      │
      └─────→ 回到观察（循环）
```

### 1. Observe 观察
收集信息，感知环境变化。
- 数据、市场信号、竞争对手动态
- **关键：看到什么，不看什么**
- 避免：信息过载、确认偏误

### 2. Orient 判断（最重要）
**博伊德认为这是整个循环的关键**——用你的心智模型、文化背景和过往经验来解读观察到的东西。
- 这个信号意味着什么？
- 和我的预期一致吗？如果不一致，为什么？
- **关键：快速更新心智模型**
- 避免：固守旧框架、分析瘫痪

### 3. Decide 决策
基于判断选择行动方案。
- 不追求最优，追求"够快够好"
- 可以同时准备多个选项
- **关键：在信息不完整时果断决策**
- 避免：无限推迟、完美主义

### 4. Act 行动
执行决策，同时观察结果。
- 行动本身就是新的信息源
- **关键：行动速度 > 计划完美度**
- 避免：只计划不执行

## 核心思想

> **速度压倒一切。** 与其花长时间做"完美"决策，不如快速循环迭代。

| | 慢 OODA | 快 OODA |
|---|---------|---------|
| 节奏 | 观察很久才行动 | 快速循环多次 |
| 信息 | 追求完整信息 | 用不完整信息快速试 |
| 对手 | 被对手牵着走 | 让对手跟不上你的节奏 |
| 结果 | 总是在反应 | 总是在引导 |

## 实际应用

| 场景 | OODA 应用 |
|------|----------|
| **创业** | 快速 MVP → 用户反馈 → 迭代（精益创业就是 OODA） |
| **竞争** | 比对手更快发现市场变化并响应 |
| **危机处理** | 快速收集信息→判断→决策→行动，不要等"全部搞清楚" |
| **个人决策** | 遇到难题先行动测试，不要陷入分析瘫痪 |

## 注意事项

- Orient 阶段的质量决定了后续所有阶段——心智模型如果错了，后面全错
- "快"不等于"鲁莽"——是快速但经过判断的
- 和 PDCA 的区别：OODA 强调速度和竞争，PDCA 强调质量和改进
- OODA 是战斗节奏，PDCA 是质量节奏——不同场景用不同循环

## 与其他模型的关系

- **PDCA**：OODA 偏竞争速度，PDCA 偏质量改进
- **精益创业**：Build-Measure-Learn 就是 OODA 的商业版本
- **第一性原理**：帮助改善 Orient 阶段的心智模型质量
- **二阶思维**：在 Orient 阶段推演连锁后果
