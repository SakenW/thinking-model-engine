# GTD (Getting Things Done)

## 概述

戴维·艾伦 (David Allen) 提出的**个人效率管理系统**，核心理念：**把所有事情从脑子里拿出来，放到可信的外部系统中。**

## 五大步骤（工作流）

```
        ┌──────────┐
        │  收集     │ ← 把一切装进"收件箱"
        │  Capture  │
        └─────┬────┘
              ▼
        ┌──────────┐
        │  澄清     │ ← 这是什么？要做什么？
        │  Clarify  │
        └─────┬────┘
              ▼
        ┌──────────┐
        │  组织     │ ← 放到该放的地方
        │  Organize │
        └─────┬────┘
              ▼
        ┌──────────┐
        │  回顾     │ ← 定期检查和更新
        │  Reflect  │
        └─────┬────┘
              ▼
        ┌──────────┐
        │  执行     │ ← 做事！
        │  Engage   │
        └──────────┘
```

### 步骤 1: 收集 (Capture)
- 把脑子里所有事情都写下来 — 任务、想法、待办、担忧
- **收件箱是唯一的入口**
- 不要分类，先全部倒出来

### 步骤 2: 澄清 (Clarify)
对收件箱的每一项问：

| 问题 | 是 → | 否 → |
|------|------|------|
| 可执行吗？ | 进入下一步 | 以后可能 / 参考 / 垃圾 |
| < 2 分钟？ | **立刻做** | 下一步 |
| 是项目吗？ | 列入项目清单，定义下一步 | 列入任务清单 |
| 指定日/委托/等待？ | 日历 / 等待清单 | 下一步行动清单 |

### 步骤 3: 组织 (Organize)
把澄清后的内容放入对应容器：

| 容器 | 内容 |
|------|------|
| 项目清单 | 需要多步完成的目标 |
| 下一步行动清单 | 单步可执行的任务（按场景分：@电脑/@电话/@外出）|
| 日历 | 有具体时间的事（硬性承诺）|
| 等待清单 | 委托出去正在等结果的事 |
| 参考资料 | 将来可能用到的信息 |
| 也许/某天 (Someday/Maybe) | 想做但不是现在 |

### 步骤 4: 回顾 (Reflect)
- **每周回顾**（GTD 的灵魂！）：
  - 清空收件箱
  - 复审项目清单和下一步行动
  - 更新日历
  - 回顾"也许/某天"清单

### 步骤 5: 执行 (Engage)
根据上下文、时间、精力选择做什么：

| 维度 | 选择依据 |
|------|----------|
| 上下文 | 现在能做什么？（在电脑前？有电话？）|
| 时间 | 有多少时间可用？|
| 精力 | 现在精力水平如何？|
| 优先级 | 哪件事影响最大？|

## 核心原则

1. **大脑是用来思考的，不是用来记事的** — 用外部系统承载记忆
2. **信任你的系统** — 如果你不相信系统会提醒你，你还是会用脑子记
3. **每周回顾不可跳过** — 这是系统不崩溃的关键

## 与其他模型的关系

| 模型 | 关系 |
|------|------|
| **两分钟规则** | GTD 澄清步骤的核心规则 |
| **艾森豪威尔矩阵** | GTD 执行步骤的优先级工具 |
| **时间阻塞** | GTD 日历的具体排程方法 |
| **帕金森定律** | GTD 通过截止日期对抗任务膨胀 |
| **心流理论** | GTD 减少认知负荷，更容易进入心流 |

## 常见误区

| 误区 | 真相 |
|------|------|
| "GTD 就是待办清单 App" | GTD 是完整方法论，App 只是工具 |
| "收集箱要清零才算 GTD" | 收集箱定期清空即可，不必每天归零 |
| "GTD 太重了不适合日常" | 可以简化为 Capture → Do → Review 三步 |
| "GTD 只适合知识工作者" | 任何有多个并行任务的人都适用 |

## 适用场景

- 个人任务与项目管理
- 信息过载时的注意力管理
- 多项目并行的优先级控制
- 团队协作中的任务追踪
- 减少焦虑和精神负荷

## 模板

```markdown
## GTD 每周回顾清单

### 📥 收集
- [ ] 清空物理收件箱（桌面、口袋、包）
- [ ] 清空数字收件箱（邮件、IM、笔记 App）
- [ ] 大脑 dump：脑子里所有未完成的事写下来

### 🧹 澄清 & 组织
- [ ] 处理每一项收件箱内容
- [ ] 更新项目清单及下一步行动
- [ ] 更新日历（硬性承诺）
- [ ] 更新等待清单

### 🔍 回顾
- [ ] 回顾日历（过去一周 + 未来两周）
- [ ] 回顾项目清单（进度、是否还相关）
- [ ] 回顾"也许/某天"清单
- [ ] 回顾年度/季度目标（OKR 进度）

### 📋 规划下周
- [ ] 确认下周最重要的 3 件事
- [ ] 预留缓冲时间（建议 20%）
- [ ] 安排深度工作时段
```
