# 时间阻塞 (Time Blocking)

## 概述

**把日程表切成固定的时间块，每个时间块只专注做一件事。**

不是 To-Do List，而是 **Time Block（时间块）** — 把任务变成日历上的"会议"。

## 核心方法

### 基础版
1. 列出今天要做的所有事
2. 估算每件事需要的时间
3. 在日历上分配时间段
4. 到点开始，到点切换

### 进阶版：按精力状态分块

```
┌─────────────────────────────────────┐
│ 早晨 (高精力)                        │
│ ┌──────────┐ ┌──────────┐           │
│ │ 深度工作   │ │ 深度工作   │           │
│ │ (90 min) │ │ (90 min) │           │
│ └──────────┘ └──────────┘           │
├─────────────────────────────────────┤
│ 午后 (中等精力)                      │
│ ┌──────────┐ ┌──────────┐           │
│ │ 会议/协作  │ │ 设计/创意  │           │
│ │ (60 min) │ │ (60 min) │           │
│ └──────────┘ └──────────┘           │
├─────────────────────────────────────┤
│ 傍晚 (低精力)                        │
│ ┌──────────┐ ┌──────────┐           │
│ │ 邮件/行政  │ │ 学习/阅读  │           │
│ │ (30 min) │ │ (30 min) │           │
│ └──────────┘ └──────────┘           │
└─────────────────────────────────────┘
```

## 关键原则

### 1. 保护深度工作时间块
- 关闭通知、手机静音
- 告知同事这段时间不可打扰
- **每天至少 2 个 90 分钟的深度工作块**

### 2. 包含缓冲时间
- 不要把一天排满 — 留 20% 缓冲
- 任务几乎总是比预估时间长

### 3. 批量处理琐碎事务
- 邮件：每天 2-3 个固定时段集中处理
- 微信/消息：不要实时回复
- 行政事务：集中在低精力时段

### 4. 单线程，不并行
- 一个时间块 = 一件事
- 上下文切换成本很高（平均 23 分钟恢复）

## 与 GTD 的关系

| | 时间阻塞 | GTD |
|---|----------|-----|
| 核心 | 日历驱动 | 列表驱动 |
| 粒度 | 时间块 | 下一步行动 |
| 重点 | 什么时候做 | 做什么 |
| 适合 | 有明确日程的人 | 事务繁杂的人 |

**最佳实践：GTD 收集 + 时间阻塞安排**
