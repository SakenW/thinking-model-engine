# ADDIE 教学设计 (ADDIE Instructional Design)

## 概述

最经典的教学设计框架：**Analysis（分析）→ Design（设计）→ Development（开发）→ Implementation（实施）→ Evaluation（评估）**。无论是设计一门课程、一次培训、还是一篇教程，ADDIE 都提供了系统化的开发流程。

## 五个阶段

| 阶段 | 核心问题 | 关键动作 | 输出 |
|------|---------|---------|------|
| **A - Analysis 分析** | 学习者是谁？要学什么？ | 需求分析、学习者画像、目标设定 | 需求文档、学习目标 |
| **D - Design 设计** | 怎么教最有效？ | 内容大纲、教学方法选择、评估方式 | 课程大纲、教学策略 |
| **D - Development 开发** | 制作具体内容 | 编写教材、制作课件/PPT/视频 | 教学材料 |
| **I - Implementation 实施** | 如何交付？ | 试点、正式交付、学习者支持 | 交付的课程/培训 |
| **E - Evaluation 评估** | 效果如何？ | 收集反馈、测量学习成果、迭代 | 评估报告、改进建议 |

## 流程图

```
A 分析 → D 设计 → D 开发 → I 实施 → E 评估
                                          ↑
                                          │
                  ← ← ← 迭代改进 ← ← ← ←┘
```

## 使用步骤

1. **Analysis** — 学习者是谁？他们已经知道什么？需要学会什么？约束条件？
2. **Design** — 用什么方法教？（讲授/案例/练习/讨论）怎么评估学习效果？
3. **Development** — 写课件、录视频、设计练习题
4. **Implementation** — 先小范围试讲，收集反馈，调整后正式交付
5. **Evaluation** — 学到了吗？满意吗？下次怎么改进？

## 示例

> **设计一个"Python 入门"培训：**
>
> - **A**：受众是非技术产品经理，0 基础，目标是能用 Python 处理 Excel 数据。时间限制 4 小时。
> - **D**：方法 = 讲解 30% + 动手 70%。评估 = 完成一个小项目（读取 CSV 并生成图表）。
> - **D**：准备 Jupyter Notebook 教程 + 练习数据集 + 参考答案
> - **I**：先找 3 个 PM 试讲，发现 2 个地方太难，简化后正式培训
> - **E**：课后问卷 + 1 个月后跟踪——80% 表示在工作中用上了

## 注意事项

- Analysis 是最容易被跳过但最重要的阶段——**不了解学习者就设计不出好课**
- Evaluation 不只是最后一个阶段——**每个阶段都可以做小评估**
- ADDIE 是线性的但实际使用中**大量迭代**——不是严格的一次走完
- 敏捷版 SAM (Successive Approximation Model) 是 ADDIE 的迭代变体

## 变体/相关模型

- **SAM (Successive Approximation Model)**：ADDIE 的敏捷迭代版
- **布鲁姆分类学**：Design 阶段设定学习目标的工具
- **第一性原理**：Analysis 阶段的深度思考方法
- **费曼学习法**：Development 阶段确保内容易懂的工具
- **Kirkpatrick 评估模型**：Evaluation 阶段的四级评估框架

## 适用场景

- 企业培训课程设计
- 在线课程/知识付费产品开发
- 学校教学设计
- 技术文档/教程编写
- 用户教育/Onboarding 设计
- 任何"教别人"的场景
