# 五遍沟通法 (Five Rounds Communication)

## 概述

亚马逊/阿里等科技公司推崇的沟通方法：**重要的事情要说五遍**，确保信息传递不失真。不是简单重复五次，而是每一遍深入一层，从"布置任务"到"理解背景"到"主动思考"。

## 五遍分别说什么

| 遍数 | 发送者说 | 接收者理解到的层次 |
|------|---------|-----------------|
| **第 1 遍** | "请去做 X" | 任务是什么 |
| **第 2 遍** | "请你复述一下，你准备怎么做 X？" | 确认理解无误 |
| **第 3 遍** | "你觉得做这件事的目的是什么？会遇到什么问题？" | 理解背景和意图 |
| **第 4 遍** | "你觉得有没有更好的方法？或者需要什么资源？" | 主动思考和承担 |
| **第 5 遍** | "如果是你来负责这件事，你会怎么规划？你有什么建议？" | 所有权转移 |

## 核心逻辑

```
第 1 遍：交待任务    → "做什么"
第 2 遍：确认理解    → "做对了吗？"
第 3 遍：理解意图    → "为什么做？"
第 4 遍：主动思考    → "怎么做更好？"
第 5 遍：承担责任    → "如果是你，你怎么规划？"
```

## 使用步骤

1. **清晰交待任务** — 说清楚做什么、交付物、截止时间
2. **要求对方复述** — 不是背诵，是用自己的话解释
3. **追问目的和风险** — 确认对方理解"为什么"
4. **鼓励提出优化** — 给对方思考和改进的空间
5. **让对方自主规划** — 从"执行者"变成"负责人"

## 示例

> **Leader 对下属：**
>
> - **L1**：下周三之前把竞品分析报告给我
> - **L2**：你复述一下——什么报告、包含什么内容、什么时候交？
> - **L3**：做这个竞品分析的目的是什么？你觉得最可能遇到什么困难？
> - **L4**：你觉得除了我说的这些维度，还需要分析什么？需要谁配合？
> - **L5**：如果完全交给你来负责，你会怎么安排这 5 天？

## 注意事项

- **不是所有事都需要五遍**——日常小事一遍就够
- 适用于**重要任务、新人带教、跨部门协作**
- 第五遍不是真的放手不管——而是让执行者有**主人翁意识**
- 这不是"考验"——而是**降低沟通成本的保险**

## 变体/相关模型

- **闭环沟通 (Closed-Loop Communication)**：医疗/航空领域的确认机制
- **5W1H**：第一遍任务交代的结构化工具
- **非暴力沟通 (NVC)**：更深层的沟通模型
- **SMART 原则**：任务交代的清晰度标准
- **RACI 矩阵**：责任分配的工具

## 适用场景

- 向下属布置重要任务
- 跨部门协作需求对齐
- 新人 Onboarding
- 项目启动会
- 客户需求确认
- 任何"沟通成本高"的场景
