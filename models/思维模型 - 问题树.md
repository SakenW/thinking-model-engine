# 问题树（逻辑树）(Issue Tree / Logic Tree)

## 概述

麦肯锡核心工具之一：把一个复杂问题**层层拆解**成可执行的子问题，像树干分叉一样。确保拆解过程遵循 MECE 原则（不重不漏），最终每个"叶子"都是一个可以直接回答或执行的具体问题。

## 结构示意

```
核心问题
├── 维度 A
│   ├── 子问题 A1
│   │   ├── 叶子 A1a（可执行）
│   │   └── 叶子 A1b（可执行）
│   └── 子问题 A2
│       └── 叶子 A2a（可执行）
├── 维度 B
│   ├── 子问题 B1
│   └── 子问题 B2
└── 维度 C
    └── ...
```

## 三种常见类型

| 类型 | 拆解方式 | 适用场景 |
|------|---------|---------|
| **议题树** | "有哪些可能？" | 寻找方案、探索机会 |
| **假设树** | "如果假设成立，需要验证什么？" | 验证假设、快速排除 |
| **决策树** | "如果选 A 会怎样？选 B 呢？" | 方案对比、风险评估 |

## 使用步骤

1. **定义核心问题** — 一句话，明确不含糊
2. **选择拆解维度** — 用什么角度切？（时间/流程/对象/因果…）
3. **逐层展开** — 每层保持 MECE
4. **判断是否足够细** — 叶子节点是否可以直接回答/执行？
5. **分配与执行** — 每个叶子分配责任人

## 示例

> **问题：公司营收下降了 20%**
>
> ```
> 营收下降 20%
> ├── 客户数量变化？
> │   ├── 新客户减少？ → 获客渠道分析
> │   └── 老客户流失？ → 流失率分析
> ├── 客单价变化？
> │   ├── 降价了？ → 竞品价格分析
> │   └── 客户买的少了？ → 复购率分析
> └── 产品结构变化？
>     ├── 高毛利产品占比下降？
>     └── 产品线收缩？
> ```

## 注意事项

- 拆到 3 层以内为宜，太深反而迷失
- 不是所有分支都同等重要——**先砍明显不相关的**
- 假设树比议题树更快——先有假设再验证，而非穷举所有可能
- 配合 5 Whys 使用效果更好——逻辑树找"面"，5 Whys 找"点"

## 变体/相关模型

- **MECE 原则**：拆解的质量标准
- **5 Whys**：深度挖掘根因
- **金字塔原理**：逻辑树的结构化表达
- **鱼骨图 (Fishbone Diagram)**：因果关系的图形化版本
- **决策矩阵**：叶子节点的方案选择工具

## 适用场景

- 复杂问题拆解
- 战略分析框架搭建
- 咨询项目问题结构化
- 根因分析
- 项目计划 WBS 分解
- 面试中的结构化表达
