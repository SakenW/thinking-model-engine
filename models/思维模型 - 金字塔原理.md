# 金字塔原理 (Pyramid Principle)

## 概述

芭芭拉·明托（McKinsey 顾问）提出的**结构化思维与表达框架**。核心：**结论先行，以上统下**。

## 四大原则

### 1. 结论先行 (Top Down)
- 最核心的观点/结论放在最前面
- 支撑信息排在后面
- 听众/读者 30 秒内知道你要说什么

### 2. 以上统下 (Summarize)
- 上层论点是下层论据的总结
- 每一层都回答"所以呢？"的问题
- 不允许出现"悬挂"的论据

### 3. 归类分组 (Group)
- 同一组的思想有逻辑关系
- 使用 MECE 原则确保不重不漏
- 常见逻辑顺序：时间、结构、重要性

### 4. 逻辑递进 (Order)
- 每组内按明确顺序排列
- 演绎推理（大前提→小前提→结论）
- 或归纳推理（并列事实→共同结论）

## 结构示意

```
         ┌───────────┐
         │  核心结论  │  ← 一句话说清楚
         └─────┬─────┘
    ┌──────┬────┴────┬──────┐
    ▼      ▼         ▼      ▼
 ┌──────┐┌──────┐┌──────┐┌──────┐
 │ 论点A ││ 论点B ││ 论点C ││ 论点D │
 └──┬───┘└──┬───┘└──┬───┘└──┬───┘
    ▼       ▼       ▼       ▼
  [证据]   [证据]   [证据]   [证据]
```

## SCQA 叙事模型（金字塔的入口）

| 阶段 | 内容 | 作用 |
|------|------|------|
| **S**ituation | 大家都认可的背景 | 建立共鸣 |
| **C**omplication | 出现了什么冲突/变化 | 制造张力 |
| **Q**uestion | 因此产生的问题 | 引发思考 |
| **A**nswer | 你的解决方案/结论 | 给出方向 |

## 实用场景

- **写邮件/报告**：开头给结论，后面展开
- **做 PPT**：每页标题就是结论
- **开会发言**：先说建议，再解释原因
- **面试回答**：先给结论再展开论证

## 注意事项

- **结论先行不等于没有论证**——结论是入口，后面必须有支撑
- **MECE 是金字塔的骨架**——分组时务必检查是否不重不漏
- 自上而下（给结论再展开）适合**汇报/写作**，自下而上（先归纳再提炼）适合**分析/研究**
- 避免"伪金字塔"——论点之间没有逻辑关系，只是形式上分层
- 一页 PPT 一个核心观点，不要塞太多信息

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 用"首先…其次…最后"堆砌 | 每组有明确的归类逻辑 |
| 结论太模糊（"我们需要提升"） | 结论具体可操作（"Q3 将 X 提升 20%"） |
| 论据不支持论点 | 每个论点下至少有一个数据或事实 |
| 层级超过 3 层 | 超过 3 层说明需要进一步概括 |

## 变体/相关模型

- **SCQA 结构表达**：金字塔的叙事入口（Situation → Complication → Question → Answer）
- **PREP 结构**：金字塔的单点精简版（Point → Reason → Example → Point）
- **MECE 原则**：金字塔分组的黄金法则
- **STAR 法则**：金字塔在案例表达中的应用
- **Minto Pyramid 的逻辑树**：演绎推理 vs 归纳推理的选择

## 适用场景

- 商业报告 / 咨询方案撰写
- PPT 结构设计
- 会议发言与汇报
- 面试回答（结构化表达）
- 学术论文框架搭建
- 邮件 / 文案写作
- 复杂问题的分析与拆解
