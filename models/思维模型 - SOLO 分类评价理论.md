# SOLO 分类评价理论 (SOLO Taxonomy)

## 概述

John Biggs 提出的学习成果分类框架：**SOLO = Structure of the Observed Learning Outcome（可观察的学习成果结构）**。从"什么都不知道"到"融会贯通"分为 5 个层次，帮助你判断自己的理解深度到底在哪。

## 五个层次

| 层次 | 名称 | 特征 | 回答"气候变化"的表现 |
|------|------|------|-------------------|
| **前结构** | Pre-structural | 没有理解，答非所问 | "气候变化就是天气不好" |
| **单点结构** | Uni-structural | 只抓住一个点 | "气候变化是因为二氧化碳多了" |
| **多点结构** | Multi-structural | 知道多个点但互不关联 | "二氧化碳多了，冰川融化，海平面上升" |
| **关联结构** | Relational | 多个点之间建立了联系 | "工业化→CO2↑→温室效应→冰川融化→海平面↑→生态失衡，这是一个因果链" |
| **抽象扩展** | Extended Abstract | 融会贯通并能迁移到新情境 | "气候变化和经济发展的矛盾可以通过碳交易机制来平衡，这个思路也适用于其他全球性公共问题" |

## 层级图

```
深度 ▲
     │  ⑤ 抽象扩展（融会贯通 + 迁移创新）
     │  ④ 关联结构（多点了然 + 建立联系）  ← 学习目标
     │  ③ 多点结构（知道很多但碎片化）    ← 大多数人停在这
     │  ② 单点结构（只知道一个点）
     │  ① 前结构（完全不理解）
     └──────────────────────────→ 广度
```

## 使用步骤

1. **评估当前层次** — 我对这个主题的理解在哪个层次？
2. **识别提升路径** — 从当前层次到上一层的差距是什么？
3. **刻意练习跨越**：
   - 从 ②→③：多收集信息点
   - 从 ③→④：**画关系图，找连接**
   - 从 ④→⑤：**跨领域类比，找应用场景**
4. **用费曼法验证** — 能用简单语言把关联说清楚 = 到了 ④

## 示例

> **学"商业模式画布"的 SOLO 进化：**
>
> - ②：知道有 9 个模块
> - ③：能列出 9 个模块的名称
> - ④：能分析一个具体公司的 9 个模块并解释它们之间的关系（如：客户细分如何影响收入来源）
> - ⑤：能用画布框架设计自己的商业模式，并发现传统行业的创新机会

## 注意事项

- 不是所有知识都需要到 ⑤——**日常够用即可的不必强求**
- ③→④ 是性价比最高的跨越——**从"知道很多"到"连成体系"**
- 考试往往只考到 ③——但真正的理解和应用需要 ④+
- 评估别人的理解层次时，**看他能否建立联系**（区分 ③ 和 ④ 的关键）

## 变体/相关模型

- **布鲁姆分类学 (Bloom's Taxonomy)**：类似的认知层次框架（记忆→理解→应用→分析→评价→创造）
- **费曼学习法**：检验和提升 SOLO 层次的工具
- **概念图 (Concept Map)**：从 ③ 提升到 ④ 的可视化工具
- **达克效应**：处于 ②③ 的人最容易高估自己

## 适用场景

- 自我学习深度评估
- 课程/教学设计
- 考试题设计（覆盖不同层次）
- 面试问题设计
- 知识管理体系（标注每条笔记的 SOLO 层次）
- 团队能力评估
