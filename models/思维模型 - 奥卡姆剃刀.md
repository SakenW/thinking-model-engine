# 奥卡姆剃刀 (Occam's Razor)

## 概述

14 世纪哲学家奥卡姆的威廉提出的原则：**如无必要，勿增实体**。在多个解释都能说明同一现象时，选择假设最少的那个。不是"最简单的总是对的"，而是"简单的应该优先检验"。

## 核心原则

| 原则 | 说明 |
|------|------|
| **简约优先** | 同等解释力下，假设越少越好 |
| **剃刀不等于真理** | 简单的解释可能错，但它是最经济的起点 |
| **区分"简单"和"简化"** | 好的简约是去掉不必要的复杂，而非回避必要的细节 |

## 使用步骤

1. **列出所有可能的解释**
2. **统计每个解释需要的假设数量**
3. **排除需要额外假设的选项**
4. **用最简解释做第一验证**
5. **如果被证伪，再考虑更复杂的**

## 示例

> **网站加载慢，排查原因：**
>
> - 解释 A：图片没压缩（1 个假设）✅ 先查这个
> - 解释 B：CDN 被攻击 + DNS 劫持 + 服务器过载（3 个假设）
>
> 用奥卡姆剃刀 → 先验证 A。A 不对再考虑 B。

## 注意事项

- **不要滥用**——世界本身就是复杂的，有些问题确实需要复杂解释
- "最简单"指的是**假设最少**，不是最容易理解
- 它是**启发式工具**，不是逻辑证明
- 科学史上很多反直觉的发现（如相对论）最初都违反奥卡姆直觉

## 变体/相关模型

- **汉隆剃刀 (Hanlon's Razor)**："不要把能用愚蠢解释的事归咎于恶意"
- **希克姆格言 (Hickam's Dictum)**：奥卡姆的反面——"病人可以同时得多种病"
- **第一性原理**：拆到最基本的不可再简的事实
- **MECE 原则**：拆解问题的结构化方法

## 适用场景

- 故障排查与 Debug
- 假设生成与筛选
- 产品功能设计（避免过度设计）
- 学术论文审稿（评估解释的必要性）
- 日常决策（在多个可行方案中选）

## 模板

```markdown
## 问题：_
## 可能解释（按假设数排序）
1. [ ] _ （_ 个假设）
2. [ ] _ （_ 个假设）
3. [ ] _ （_ 个假设）

## 验证顺序 & 结果
_
```
