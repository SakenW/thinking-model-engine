# 福格行为模型 (Fogg Behavior Model)

## 概述

斯坦福大学 B.J. Fogg 提出的行为设计公式：**B = MAP**。行为 (Behavior) = 动机 (Motivation) × 能力 (Ability) × 提示 (Prompt)。三个要素同时满足，行为才会发生。

## 核心公式

```
行为 = 动机 × 能力 × 提示 (B = MAP)

如果任何一项为 0 → 行为不会发生：
- 有动机 + 有能力 + 没提示 → 忘了做
- 有动机 + 没能力 + 有提示 → 做不了
- 没动机 + 有能力 + 有提示 → 不想做
- 三个都有 → ✅ 行为发生！
```

## 三个要素

| 要素 | 说明 | 关键策略 |
|------|------|---------|
| **M - Motivation 动机** | 想做吗？ | 短期：利用趋利/避害/社交/希望/恐惧 |
| **A - Ability 能力** | 做得到吗？ | **降低门槛是最高效的方法**——简化步骤、减少时间/金钱/体力/脑力消耗 |
| **P - Prompt 提示** | 想起来了吗？ | 锚定在已有行为之后（"刷牙后"→"做俯卧撑"） |

## 行为轴模型

```
        动机高
           ▲
           │    ┌──────────────┐
           │    │  在这里放提示  │ ← 容易发生
           │    │  行为很可能会  │
           │    │  发生         │
           │    └──────────────┘
           │
           │    ┌──────────────┐
           │    │  需要提高动机  │ ← 行动线
           │    │  或降低难度    │
           │────┴──────────────┴────→ 能力强→弱（行为越容易做）
           │         动机低
```

## 使用步骤

### 想让人做某事（如：培养好习惯）

1. **明确目标行为** — 具体、可观测（不是"更健康"，是"每天走 5000 步"）
2. **降低门槛（A）** — 怎么让这件事变得超简单？（哪怕只做 1 分钟也算）
3. **设计提示（P）** — 把它锚定在一个已有习惯之后
4. **提升动机（M）** — 找到内在动机或设计外部激励

### 想让人不做某事（如：戒掉坏习惯）

1. **提高门槛（A）** — 增加步骤、增加阻力
2. **移除提示（P）** — 看不到、摸不到、想不到
3. **降低动机（M）** — 让它变得不吸引人

## 示例

> **培养"每天冥想 5 分钟"习惯：**
>
> - **M**：知道冥想能减轻焦虑（动机已足够）
> - **A**：门槛太高？→ 把目标降到"每天打开冥想 App 就算成功"（先建立习惯，再增加时长）
> - **P**：提示 = "每天早上刷完牙之后"
>
> **减少刷短视频：**
> - **P**：把 App 图标藏到第二页 + 关闭通知
> - **A**：设置每天 30 分钟屏幕使用限额
> - **M**：睡前读一本有趣的书替代（更想做别的事）

## 注意事项

- **改变能力（降低门槛）比改变动机更容易**——这是 Fogg 的核心洞察
- "从小开始"不是妥协——**微习惯是建立习惯的最可靠路径**
- 提示必须是**具体的时间和情境**——"有空时"不是提示，"吃完午饭后"才是

## 变体/相关模型

- **Hook 上瘾模型**：Fogg 模型在产品中的商业应用
- **习惯回路 (Habit Loop)**：提示→惯常行为→奖赏（Charles Duhigg）
- **WOOP**：更结构化的行为规划工具
- **心流理论**：高动机+高能力时的理想状态
- **行为经济学 (Nudge)**：通过"助推"改变选择的设计方法

## 适用场景

- 习惯养成/戒除
- 产品用户行为设计
- 健康管理（运动、饮食、睡眠）
- 学习计划执行
- 团队行为改变
- 自我管理工具设计
