# PDCA 循环

## 概述

**计划 → 执行 → 检查 → 行动**，持续改进的闭环框架。

由休哈特提出，戴明推广，也称**戴明环 (Deming Cycle)**。

## 四个阶段

```
        ┌─────────── Plan（计划）
       │    确定目标和方案
       │
       ▼
  Act ←┘  Do（执行）
  │     按计划实施
  │
  ▼
Check   Check（检查）
│       评估结果 vs 目标
│
└──────→ （进入下一轮）
```

### P - Plan（计划）
- 定义问题/目标
- 分析现状，找出根因（可用 5 Whys）
- 制定行动计划：做什么、谁做、何时完成、怎么衡量

### D - Do（执行）— 小规模试点
- **先小范围执行**，不要全面铺开
- 记录过程中的观察和数据
- 收集反馈

### C - Check（检查）
- 对比实际结果与预期目标
- 分析差异原因
- 总结学到了什么

### A - Act（行动 / 处理）
- **成功** → 标准化、推广到更大范围
- **部分成功** → 调整后进入下一轮 PDCA
- **失败** → 总结教训，换方案重来

## 关键原则

1. **每轮都是螺旋上升** — 不是原地转圈
2. **先试点再推广** — 不要一上来就 All-in
3. **数据驱动检查** — 用事实说话，不靠感觉
4. **标准化成功经验** — 否则每次都从零开始

## 适用场景

- 质量改进（ISO 9001 核心方法）
- 项目迭代（敏捷开发的底层逻辑）
- 个人习惯养成
- 流程优化
- 问题解决

## 与敏捷开发的关系

| PDCA | 敏捷对应 |
|------|----------|
| P | Sprint Planning |
| D | Sprint 执行 |
| C | Sprint Review + Retrospective |
| A | Backlog 调整 + 下 Sprint |

## 变体

- **PDSA**（Study 替代 Check）：更强调"学习"而不只是"检查"
- **SDCA**（Standardize）：用于维持已有标准
- **OPDCA**：加 O (Observe) 观察，在 P 之前先了解现状
