# 价值主张画布 (Value Proposition Canvas)

## 概述

商业模式画布中"价值主张"模块的**深度展开版**，用于精确匹配**产品/服务**与**客户需求**。

由 Alexander Osterwalder 设计，通常与商业模式画布配合使用。

## 两大组成部分

```
┌───────────────────┐     ┌───────────────────┐
│    客户画像        │     │   价值地图         │
│                    │     │                   │
│  ┌──────────────┐  │     │  ┌──────────────┐  │
│  │   客户任务    │◄─┼────►││  产品与服务    │  │
│  │  (Jobs)      │  │     │  │              │  │
│  ├──────────────┤  │     │  ├──────────────┤  │
│  │   客户痛点    │◄─┼────►││  痛点缓释方案  │  │
│  │  (Pains)     │  │     │  │              │  │
│  ├──────────────┤  │     │  ├──────────────┤  │
│  │   客户收益    │◄─┼────►││  收益创造方案  │  │
│  │  (Gains)     │  │     │  │              │  │
│  └──────────────┘  │     │  └──────────────┘  │
└───────────────────┘     └───────────────────┘
       客户侧                       产品侧
         ↕           Fit（匹配）
```

## 一、客户画像（右侧）

### 客户任务 (Customer Jobs)
客户试图完成的事情：
- **功能性任务**：解决具体问题、完成任务
- **社会性任务**：在群体中获得影响力/地位
- **情感性任务**：获得安全感、成就感
- **支持性任务**：作为买家/用户的相关工作（如比价、安装）

### 客户痛点 (Pain Points)
- 不好的结果 / 障碍 / 风险
- **痛点等级**：一般痛点 → 强烈痛点 → 极度痛点（→ 机会！）

### 客户收益 (Gain Points)
- 想要的结果 / 超出预期的惊喜
- **必需收益**（基本要求）vs **期望收益**（想要但非必须）vs **惊喜收益**

## 二、价值地图（左侧）

### 产品与服务
你提供什么来帮助客户完成任务？

### 痛点缓释方案
- 消除坏结果
- 降低障碍
- 减少风险

### 收益创造方案
- 创造预期收益
- 创造超出预期的收益
- 支撑决策/采用

## 核心问题：Fit（匹配）

> **你的产品是否真的解决了客户的痛点？是否真的创造了他们想要的收益？**

- 没有 Fit → 产品没人要
- 部分 Fit → 有机会优化
- 强 Fit → PMF (Product-Market Fit)

## 使用步骤

1. 先做**客户画像**（从客户视角出发）
2. 再做**价值地图**（从产品视角出发）
3. 找**匹配点** — 用连线标注哪些产品特性对应哪些客户需求
4. **识别缺口** — 哪些痛点没被覆盖？哪些收益没被创造？
5. 迭代产品或重新定义目标客户
